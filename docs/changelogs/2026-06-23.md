# 2026-06-23 更新日志

## 修复

### Nginx 崩溃 (Exited 255)
- **根因**: `nginx.conf` 中 `minio:9000` 和 `weclaw-st-*:8000` 的 `proxy_pass` 使用硬编码 hostname，在目标容器就绪前 nginx 解析 DNS 失败导致 emerg 退出
- **修复**: 改为变量方式 (与 api-server/dashboard 一致)，利用 Docker DNS resolver 延迟解析
  ```
  # 之前: proxy_pass http://minio:9000/;
  # 之后: set $minio_backend "http://minio:9000"; proxy_pass $minio_backend;
  ```

### ChatBridge WebSocket 连通
- **根因**: ChatBridge 扩展 `index.html` 硬编码 `ws://172.18.0.1:8850`，Docker bridge IP 在 WSL2 下从 Windows 浏览器不可路由
- **修复**: 改为 `ws://localhost:8850`，利用 WSL2 localhost 端口转发
- **持久化**: ChatBridge 文件从 git submodule 转为普通文件跟踪；bind mount 源路径已同步修复文件

### 清理
- 删除 7 个遗留孤儿容器 (bull-worker, bridge, fastagent 等)
